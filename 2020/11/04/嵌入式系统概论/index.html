2020秋 嵌入式系统概论

一 嵌入式系统简介#

嵌入式系统定义**#

IEEE：用于控制、监视或者辅助操作机器和设备的装置

国内普遍认同的定义：嵌入式系统是 “以应用为中心,以计算机技术为基础,软硬件可裁减,适用于应用系统对功能、可靠性、成本、体积、功耗有严格要求的专用计算机系统”

嵌入式系统特点**#

  1. 形式多样，面向特定用途；
  2. 得到多种处理器和处理器架构支持；
  3. 嵌入式系统通常非常注重成本问题；
  4. 有实时性和可靠性的要求；
  5. 需要专门的方法和开发工具；
  6. 嵌入式系统使用的操作系统一般是适应多种处理器、可剪裁、轻量型、实时可靠、 可固化的嵌入式操作系统。

嵌入式系统分类**#

  • 按嵌入式处理器的位数来分类
  • 按应用来分类
  • 按速度分类
    • 强实时系统
    • 一般实时系统
    • 弱实时系统
  • 按确定性来分类
    • 硬实时
    • 软实时
  • 按嵌入式系统软件复杂程度来分类
    • 循环轮询系统
    • 有限状态机系统
    • 前后台系统
    • 单处理器多任务系统
    • 多处理器多任务系统

嵌入式系统应用（按照应用分类）**#

  • 信息家电
  • 移动终端
  • 工业控制
  • 汽车电子
  • 通信类
  • 可穿戴设备
  • 航天航空

嵌入式系统组成**#

  • 一般是由嵌入式硬件和软件组成的
  • 硬件包括以微处理器为核心集成存储器和系统专用的IO设备
  • 软件包括初始化代码及驱动，嵌入式操作系统和应用程序等
    结构图 应用软件-中间件-实时内核-驱动-系统硬件

IOT**#

IOT（物联网）是万物互联的互联网，是互联网和嵌入式系统发展到高级阶段的融合。分为感知层、传输层、应用层。

CPS**#

信息物理系统，计算进程和物理进程的统一体，集计算、通信、控制于一体的下一代智能系统

基本组成 传感器 执行器 决策控制单元

通用计算机与嵌入式计算机对比#

特征 通用计算机 嵌入式计算机

组成 软件和硬件相对独立 软件与硬件是紧密集成在一起的

二 嵌入式系统设计#

嵌入式系统面临挑战#

  1. 需要多少硬件?
  2. 如何满足时限要求,如何处理多项功能在时间上的协调一致关系?
  3. 如何降低系统的功耗?
  4. 如何设计以保证系统可升级?
  5. 如何保证系统可靠地工作?

嵌入式系统的设计过程的基本流程#

  • 需求分析
  • 规格说明
  • 体系结构
  • 构件设计
  • 系统集成

传统开发过程#

  • 基本特征
    • 系统从一开始就被划分为软件和硬件两大部分
    • 软件和硬件独立进行开发设计
    • Hardware first approach often adopted
  • 隐含问题
    • 软硬件之间的交互受到很大限制
    • 系统集成相对滞后，NRE（Non-Recurring Engineering）较大

软硬件划分#

  • 软件硬件划分通常由速度、灵活性以及开销来决策
  • 嵌⼊式系统的设计涉及硬件与软件部件,设计中必须决定什么功能由硬件实现,什么功能由软件实现。
  • 硬件和软件具有双重性
  • 软硬件变动对系统的决策造成影响
  • 划分和选择需要考虑多种因素
  • 硬件和软件的双重性是划分决策的前提

通常由软件实现的部分#

  • 操作系统功能
    • 任务调度
    • 资源管理
    • 设备驱动
  • 协议栈
    • TCP／IP
  • 应用软件框架
  • 除基本系统、物理接口、基本逻辑电路，许多由硬件实现的功能都可以由软件实现。

双重性部分#

  • 算法(加密/解密，编码/解码，压缩/解缩)
  • 数学运算(浮点运算)

三 嵌入式微处理器#

冯诺伊曼结构与哈佛结构#

冯诺依曼结构将指令和数据存放在 同一存储空间 中， 统一编址指令和数据通过 同一总线访问

哈佛结构则是不同于冯诺依曼结构的一种 并行体系结构 ，其主要特点是程序和数据存储在 不同的存储空间 中，即程序存储器和数据存储器是两个相互独立的存储器，每个存储器独立编制、独立访问 。与之相对应的是系统中设置的 两条总线 （程序总线和数据总线）

RISC和CISC#

  • Complex
    • 很多寻址模式
    • 很多操作
  • Reduced
    • load/store
    • 流水线指令
CISC RISC
指令系统 复杂，庞大 简单，精简
指令字长 不固定 等长
访存指令 不加限制 只有LOAD/STORE
指令执行周期 相差很大 通常在一个周期内完成
寄存器数目 寄存器较少 寄存器较多

大端与小端#

操作系统内容，在此不赘述 （小端——数据的低字节保存在内存的低地址

嵌入式微处理器分类#

  • 嵌入式微处理单元（MPU）
  • 嵌入式微控制器（MCU）
  • 嵌入式DSP处理器——信号处理
  • 嵌入式SoC

嵌入式微处理器特点#

  • 体积小、重量轻、可靠性高
  • 功耗低
  • 成本低
  • 工作温度、抗电磁干扰、可靠性灯方面增强

WatchDog*#

看门狗是一个可以在一定时间内被复位的计数器。

实现嵌入式系统在无人状态下的连续工作。

当看门狗启动后，计数器开始自动计数，经过一定时间，如果没有被复位，计数器溢出就会对CPU产生一个复位信号使系统重启（俗称“被狗咬”）。

系统正常运行时，需要在看门狗允许的时间间隔内对看门狗计数器清零（俗称“喂狗”），不让复位信号产生。如果系统不出问题，程序保证按时“喂狗”，一旦程序跑飞，没有“喂狗”，系统“被咬”复位。

五 嵌入式系统总线*#

总线结构，常见的总线及特点*#

单总线——单一的系统总线 连接CPU 主存和 I/O设备

双总线——CPU 和 主存 之间专门设置一组高速的存储总线 CPU可通过专用总线于存储器交换信息，并减轻了系统总线的负担

多总线——在双总线的基础上增加I/O总线，其中，系统总线是 CPU 主存和通道（IOP） 之间进行数据传送的公共通路

忙等IO和中断IO*#

忙等*#

缺点：效率低，CPU 在测试设备时候不能工作，难以IO 并发

中断*#

中断：让设备可以改变CPU 的控制流，需要子例程去处理设备

中断行为：基于子例程调用机制
中断使得下一个指令是预先确定好的子例程调用
返回地址预先保存，为了返回执行前台程序

中断过程：CPU响应中断–>设备发送中断向量 –>CPU调用处理函数 –>软件处理请求 –>CPU把状态返回到前台程序

中断处理过程：中断请求、中断排队或优先级仲裁、中断处理、中断返回

使中断更具体的机制：

  1. 优先级：哪个中断得到CPU
  2. 向量机制，哪个代码是那种类型调用的

中断开销：中断延迟、寄存器现场保护、进入中断服务、中断恢复

Independent I/O port | memory-mapped I/O*#

Independent I/O port 只有特定的IO命令能访问

memory-mapped I/O 1 device control registers are just variables in memory and can be addressed in C the same way as any other variables. 2 no special protection mechanism

六 嵌入式软件*#

嵌入式软件 特点*#

  • 内存有限
  • CPU往往恰好满足要求
  • RTOS、专有OS或裸机
  • 有实时性要求（确定性）
  • 采用交叉开发的方式
  • 执行流程：多数嵌入式设备从开机开始就会运行某个程序，一直运行到系统关机。这个程序可能存储在ROM，也可能从非易失性存储器转移到RAM中执行
  • 每个嵌入式设备都是不同的
  • 需要专门的开发工具

嵌入式软件的分类*#

系统软件、支撑软件、应用软件

嵌入式软件体系结构*#

  • 轮转结构（round-robin architecture）
  • 状态机 ( State Machine )
  • 带有中断的轮转结构（ round-robin with interrupts architecture）
  • 中断（Just Interrupts）
  • 函数队列结构（function-queue-scheduling architecture）

RTOS概念**#

对外来事件能在限定的时间内做出预定质量响应的计算机系统

RTOS特点**#

  • 可靠性
  • 可预测性
  • 高性能
  • 紧凑
  • 可扩展性

RTOS选型原则**#

综合权衡

  1. 成本
  2. 可靠性
  3. 实时性
  4. 工具链
  5. 模块丰富
  6. RTOS 内核 RAM、ROM 占用量
  7. 支持

任务管理***#

实时系统及任务调度（RMS、EDF算法）

存储管理——嵌入式系统静态和动态内存管理**#

ucOS-II不划分内核和用户空间，系统只有一个地址空间。采用固定分区的存储管理方法，含有多个内存分区，不同分区中的内存块大小不同

OSMemCreate OSMemGet OSMemPut

任务的栈空间分配方法

静态分配：在编译的时候分配 static OS_STK MyTaskStack[stack_size]

动态分配：在任务运行的时候使用 malloc() 函数来动态申请内存空间

  • 在ANSI C中malloc()和free()两个函数动态地分配内存和释放内存。易产生碎片。
  • 由于内存管理算法的原因，malloc()和free()函数执行时间是不确定的。μC/OS-II 对malloc()和free()函数进行了改进，使得它们可以分配和释放固定大小的内存块。这样⼀来，malloc()和free()函数的执行时间也是固定的

七 嵌入式实时内核（结合ucOS-II和freertos）**#

实时任务调度、中断与时钟、同步与通信、存储管理（静态、动态）

实时任务调度**#

近似地每时每刻总是让优先级最高的就绪任务处于运行状态” 。为了保证这一点，它在系统或用户任务调用系统函数及执行中断服务程序结束时总是调用调度器，来确定应该运行的任务并运行它 。

μC/OS_II进行任务调度的依据就是任务就绪表

中断与时钟**#

μC/OS-II系统响应中断的过程为： 系统接收到中断请求后，这时如果CPU处于中断允许状态（即中断是开放的），系统就会中止正在运行的当前任务，而按照中断向量的指向转而去运行中断服务子程序；当中断服务子程序的运行结束后，系统将会根据情况返回到被中止的任务继续运行或者转向运行另一个具有更高优先级别的就绪任务。

同步与通信**#

管理的核心是事件控制块ECB，同步可以通过开关中断实现。互斥使用信号量。通信可以使用邮箱和消息队列实现。

存储管理（静态、动态）**#

ucOS-II不划分内核和用户空间，系统只有一个地址空间。采用固定分区的存储管理方法，含有多个内存分区，不同分区中的内存块大小不同

八 bsp, bootloader#

嵌入式系统的启动过程#

  1. 上电复位、板级初始化
  2. 系统引导/升级
  3. 操作系统初始化
  4. 应用程序初始化
  5. 多任务应用运行

BSP ，特点，与 BIOS 区别#

BSP 全称“板级支持包”(Board Support Packages),说的简单一点,就是一段启动代码, 和计算机主板的 BIOS 差不多,但提供的功能区别就相差很大

特点：硬件相连性——BSP必须为操作系统提供操作和控制具体硬件的方法；操作系统相连性——BSP必须为不同的操作系统提供接口；

区别：BIOS主要是负责在电脑开启时检测、初始化系统设备（设置栈指针，中断分配，内存初始化..）、装入操作系统并调度操作系统向硬件发出的指令。

BSP是和操作系统绑在一起运行在主板上的，尽管BSP的开始部分和BIOS所做的工作类似，可是大部分和BIOS不同，作用也完全不同。

程序员还可以编程修改BSP，在BSP中任意添加一些和系统无关的驱动或程序，甚至可以把上层开发的统统放到BSP中。 而BIOS程序是用户不能更改，编译编程的，只能对参数进行修改设置。更不会包含一些基本的硬件驱动。

引导模式#

操作系统引导概念：将操作系统装入内存并开始执行的过程。

按时间效率和空间效率不同的要求，分为两种模式：

  1. 需要BootLoader的引导模式：节省空间，牺牲时间，适用于硬件成本低，运行速度快，但启动速度相对慢
  2. 不需要BootLoader的引导模式：时间效率高，系统快速启动，直接在NOR flash或ROM系列非易失性存储介质中运行，但不满足运行速度的要求

bootloader及其启动过程#

bootloader的概念：bootloader是嵌入式系统中的OS启动加载程序。

启动过程：

  1. 初始化硬件，如设置UART（至少设置一个），检测存储器等
  2. 设置启动参数，告诉内核硬件的信息，如用哪个启动界面，波特率
  3. 跳转到操作系统的首地址
  4. 消亡
