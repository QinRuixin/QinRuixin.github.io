C/C++编译链接过程#

strace#

可使用 strace -f gcc a.c 查看 a.c 文件的编译链接过程

主要的系统调用：execve, read, write

执行程序：

  • cll 编译器（C -> 汇编）
  • as 汇编器（汇编 -> ELF relocatable）
  • collect2 收集器（收集构造函数信息）
  • ld 链接

源文件(.c/.cpp)–预处理–>纯C代码–编译(cc)–>汇编程序(.s)–汇编(as)–>目标文件(.o)–链接(ld)–>可执行文件

gcc#

用于编译、链接

  • -E：只进行预处理
  • -S：只进行预处理、编译
  • -c：预处理、编译、汇编
  • -o [output_file]：指定输出文件名
  • -g：产生符号文件（可用于调试工具）
  • -On：优化等价，n可以取0,1,2,3
  • -Wall：显示所有警告信息
  • -Idir：指定额外的头文件搜索路径
  • -Ldir：指定额外的库文件搜索路径
  • -lname：指定链接时搜索的库文件（name要去掉lib，如要链接pthread，pthread的静态库名为libpthread.a，则参数写成-lpthread）
  • -DMACROT[=DEFN]：定义宏

binutils工具集#

addr2line, 将地址转换成文件名和行号。
ar, 创建修改提取文档(制作静态库)
as, 汇编器
ld, 链接器
nm, 列出文件中符号
objdump, 反汇编
ranlib, 创建归档索引(制作静态库)
readelf, 显示ELF二进制信息
size, 显示文件中每个部分大小
strip, 文件瘦身
xxd, 查看二进制文件

ELF#

ELF, Executable and Linking Format 定义了目标文件内
部信息如何组成和组织的文件格式。
readelf -h filename
readelf -S filename

readelf 查看 ELF 文件#

readelf -h … 查看文件头

  • 文件内容的分布
  • 指令集体系结构
  • 入口地址

readelf -a 所有

readelf -l 文件头 + 如何被加载

LOAD 文件的 Offset 映射到 虚拟地址 文件地址
地址空间随机化  位置无关代码  默认偏移量是 0

ELF 的 program headers

决定了 ELF 应该如何被加载器加载（执行）

objdump#

objdump 命令用于反汇编目标文件。

objdump -d （–disassemble 反汇编） filename

objdump -d -j .text filename

objdump -g （–debugging）

gdb调试#

starti 可以从第一条指令开始执行程序

layout asm 可以查看汇编

info register 可以查看寄存器

-g选项编译: $ gdb -g -o fac fac.c
$ gdb ./fac
(gdb) list(l) (显示10行源代码)
(gdb) break(b) 10 (在第10行设置断点)
(gdb) run(r) (运行程序)
(gdb) print(p) j (显示变量值, 发现未初始化)
(gdb) continue(c) (继续)
(gdb) next(n) (单步)
(gdb) step(s) (单步)
(gdb) quit(q) (退出)

Makefile#

Makefile定义整个工程的编译规则。

make命令调用Makefile实现自动化编译。

  • 目标
  • 先决条件
  • 动作

在定义好依赖关系后，后续的那一行定义了如何生成目标文件的操作系统命令，一定要以一个Tab键作为开头。

一个简单的Makefile例子

$ make

$ make -f Makefile

预定义变量#

变量 含义
$< 第一个依赖文件的名称
$? 所有的依赖文件，以空格分开，这些依赖文件的修改日期比目标的修改日期晚
$+ 所有的依赖文件，以空格分开，并以出现的先后为序，可能包含重复的依赖文件
$^ 所有的依赖文件，以空格分开，不包含重复的依赖文件
$* 不包括扩展名的目标文件名称
$@ 目标的完整名称
$% 如果目标是归档成员，则该变量表示目标的归档成员名称
